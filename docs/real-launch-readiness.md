# Real Launch Readiness — 10/10 Checklist

This is the canonical "is it shippable to a real customer" checklist for
the estate-advisory public site, client portal, and partner portal.

It separates four buckets so reviewers can scan in seconds:

- **✅ Done** — implemented in this branch and verifiable from the code or
  a deployed preview.
- **🟡 Pending (engineering)** — work we already know how to do; just not
  in this PR.
- **🛑 Blocked by business input** — needs a real fact, file, signature,
  or credential from the operator. We will not invent these.
- **🌱 Post-launch optimization** — explicitly deferred; safe to launch
  without.

> Companion docs: [`launch-readiness-checklist.md`](./launch-readiness-checklist.md)
> covers the data-layer / RLS / idempotency side. This doc covers the
> customer-facing readiness side.

## Trust & compliance content

We deliberately did not invent any of the following. Each must be
confirmed in writing by the operator before it appears on the public
site:

- 🛑 **Years in business / number of estates handled** — no specific
  number is shown until provided.
- 🛑 **Insurance coverage** — no policy limits, carrier name, or COI
  badge until a current certificate is provided.
- 🛑 **Bond / fidelity coverage** — same.
- 🛑 **BBB rating / accreditation badge** — only if the operator is
  actually accredited.
- 🛑 **USPAP-compliant appraisal claim** — only if the on-staff appraiser
  is genuinely USPAP-compliant; a personal designation, not a company
  one.
- 🛑 **Auctioneer license number** — state-specific; only if held.
- 🛑 **Specific testimonials, names, photos** — only with signed release.
- 🛑 **Awards / press logos** — only with permission and a real source.
- 🛑 **Specific dollar outcomes attributed to a real estate** — only with
  consent.
- 🛑 **Phone number / email shown publicly** — currently we route to the
  walkthrough form and disclose direct contact after first call. Replace
  when the operator has a dedicated business line.

## Deploy / merge steps

1. Merge this PR to `main`.
2. Vercel will redeploy `auction-repo.vercel.app`.
3. To switch lead routing from "soft-fail demo" to "real":
   - Add the Supabase `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`,
     and `SUPABASE_SERVICE_ROLE_KEY` env vars on Vercel.
   - Apply migrations 0003–0007 from `supabase/migrations/`.
   - (Optional) seed sample data from `supabase/seed/`.
4. To enable analytics, set `NEXT_PUBLIC_ANALYTICS_DEBUG=true` in dev to
   confirm events fire, then add a vendor integration in
   `src/lib/analytics.ts`.
5. Before the public launch URL goes on cards / ads, the items in the
   "Trust & compliance content" 🛑 section above must be reviewed by
   the operator and counsel.
